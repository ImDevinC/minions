# Progress Log
PRD: cost-token-tracking
Started: 2026-03-24

## Codebase Patterns
**Migration pattern**: Simple ALTER TABLE statements for schema changes. Comments on columns for documentation. Explicit down migrations for rollback safety.

**Query field updates**: When adding DB columns, update in order: (1) scanMinion/scanMinionWithOwner helpers (2) all SELECT queries (3) test mock data. Comments document expected field count for easy verification.

**Token combining**: Store granular fields in DB, combine for display. SQL aggregations combine at query time (GetStats), API responses combine in-memory (HandleGet). Pattern: input = input + cache_read; output = output + reasoning + cache_write.

**Frontend formatting helpers**: control-panel uses formatCost()/formatTokens() helpers for consistent display. Cost shows 5 decimals (toFixed(5)), tokens show thousands separators (toLocaleString()).
